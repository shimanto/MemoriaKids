## 1. PDFファイル作成上の注意事項

**出典**: https://www.pcinfo.jpo.go.jp/site/3_support/3_attention/03_09_10-1.html

### 1.1 フォーマット要件

| 項目 | 要件 |
|------|------|
| **PDFバージョン** | 1.2 〜 1.7 に準拠した形式のみ |
| **用紙サイズ** | A4（21cm × 29.7cm） |
| **形式の優先度** | テキスト形式を推奨（イメージ形式は避ける） |

### 1.2 禁止事項

1. **PDF編集機能（コメント・注釈）の使用禁止**
   - コメント・注釈は表示されない可能性がある
   - 完成後のPDFに追加加工を行わない

2. **セキュリティ設定の禁止**
   - パスワード、暗号化、編集制限などを一切付けない
   - 出願ソフトで警告されないケースもあるため要注意

3. **タイトルプロパティの設定禁止**
   - PDFプロパティに「タイトル」を設定しない
   - 表示に時間がかかる原因となる

### 1.3 推奨事項

1. **フォント埋め込み**: すべてのフォントをPDFに埋め込む
2. **ファイル構成**:
   - 「筆頭物件」と「添付物件」は別々のPDFに分割
   - 複数の添付物件は物件ごとにPDFを作成
3. **大容量PDF**: Adobe社提供の圧縮ツール等を利用

### 1.4 検証方法

| 確認項目 | 操作手順 |
|----------|---------|
| **コメント有無** | PDF表示画面 → 右側のコメントボタンをクリック |
| **セキュリティ** | ファイル → プロパティ → セキュリティタブ |
| **タイトル** | ファイル → プロパティ → 概要タブ |

## 2. イメージの規定

**出典**: `img_01_イメージの規定.pdf`（SD12セクション）

### 2.1 図の内容とイメージ形式の対応

#### 特許・実用（四法）

| 図の内容 | 使用可能な形式 | カラー |
|----------|---------------|--------|
| 図面 | PNG / GIF / BMP（透過などの特殊設定なし） | モノクロ2値 |
| 図面（写真） | JPEG（JFIF形式のみ） | グレースケール（8bit / 256階調のみ） |

#### 意匠・商標

| 図の内容 | 使用可能な形式 | カラー |
|----------|---------------|--------|
| 線で描く図 | PNG / GIF / BMP | モノクロ2値 |
| 色彩を付した図 / 濃淡のある図 / カラー写真 / 白黒写真 | JPEG（JFIF形式） | フルカラー（RGB） |

### 2.2 形式別の重要な注意

#### PNG・GIF

- **モノクロ2値必須**（カラーの場合は警告→自動変換）
- 透過・透明度情報は「背景白」前提でモノクロ2値に変換される
- インターレースなど固有情報は削除される
- **アニメーション形式はエラー** → 使用禁止

#### BMP

- **必ずモノクロ2値で保存**
- Windowsの「ペイント」で「モノクロビットマップ」として保存可能

#### JPEG

- **JFIF形式のみ**（Exif形式は警告・自動変換）
- 特許・実用：8bit グレースケール
- 意匠・商標：24bit フルカラー
- プログレッシブ形式はエラー
- カラーモードがRGB以外（CMYK等）はエラー
- **JPEGは図面代用写真（顕微鏡写真）に限り推奨**
- それ以外はPNG/GIF/BMPの使用を推奨（データ容量がJPEGの1/8〜1/10）

### 2.3 解像度・画素密度

#### 特許・実用・国際出願（【図ｎ】等）

| 形式 | 解像度 | mm換算 | ドット換算 |
|------|--------|--------|-----------|
| PNG / GIF / BMP / JPEG | 200dpi | 170 × 255 mm | 1338 × 2007 |
| 〃 | 300dpi | 170 × 255 mm | 2007 × 3011 |
| 〃 | 400dpi | 170 × 255 mm | 2677 × 4015 |

> **判定ルール**: ドット数が200dpiの範囲を超えると300dpi、300dpiの範囲を超えると400dpiと判定される。

#### 意匠

| 形式 | 解像度 | サイズ |
|------|--------|--------|
| JPEG（フルカラー） | 200dpi のみ | 150 × 113 mm（1181 × 889 ドット） |
| PNG / GIF / BMP | 400dpi のみ | 150 × 113 mm（2362 × 1779 ドット） |

#### 商標

| 形式 | 解像度 | サイズ | 備考 |
|------|--------|--------|------|
| JPEG（フルカラー） | 200dpi のみ | 150 × 150 mm | 推奨：80×80mm or 150×150mm |
| PNG / GIF / BMP | 400dpi のみ | 150 × 150 mm | 〃 |

### 2.4 計算式

ドット数からサイズ（cm）を求める計算式：

```
200dpi の場合: ドット数 ÷ 200 × 2.54 ≒ 長さ（cm）
300dpi の場合: ドット数 ÷ 300 × 2.54 ≒ 長さ（cm）
400dpi の場合: ドット数 ÷ 400 × 2.54 ≒ 長さ（cm）
```

> **重要**: インターネット出願ソフトでは画像の解像度（dpi情報）は無視され、**ドット数からサイズが判定される**。

### 2.5 重要な注意点

- 図面代用写真以外の図表・線図・化学式は **PNG/GIF/BMP のモノクロ2値** を使用すること
- PDFを印刷するときは「**実際のサイズ**」を指定しないと正しいサイズで印刷されない
- 国際出願願書・国際予備審査請求書はWIPO独自レイアウト仕様のため、図が小さく印刷される場合がある
- 必ず送信前に「送信ファイル」フォルダ内の書類を印刷して内容を確認すること

## 3. イメージ組み込み可能な箇所

**出典**: `img_02_イメージ埋め込み箇所.pdf`

イメージを組み込める箇所はXML系書類とSGML系書類で異なる。

### 3.1 XML系（特許・実用・PCT国際出願）

| 標準項目 | 組み込み可能な箇所 | 制限 |
|---------|------------------|------|
| 請求項 | 請求の範囲の【請求項ｎ】【化ｎ】【数ｎ】【表ｎ】 | 【化ｎ】【数ｎ】【表ｎ】は1項目1イメージ |
| 段落番号 | 新様式明細書の【０００１】【化ｎ】【数ｎ】【表ｎ】 | 〃 |
| 配列表 | 配列表（HTML）の内容 | - |
| 要約書 | 要約書の内容 | - |
| 図 | 図面の【図ｎ】 | 1項目1イメージ |
| 段落 | 外国語明細書、外国語特許請求の範囲、外国語図面、外国語要約書 | - |
| 意見の内容 | 誤訳訂正書、意見書、上申書、弁明書、早期審査関係、優先審査関係、実用新案技術評価請求書、回復理由書 等 | - |
| 添付物件 | 限定された書類（特許協力条約第19条/34条補正の写し等） | - |
| 補正の内容 | 手続補正書、誤訳訂正書 等で上記箇所を補正する場合 | - |
| 証明に係る事項 | 証明請求書 | 請求書類 |

### 3.2 SGML系（意匠・商標）

| 標準項目 | 組み込み可能な箇所 |
|---------|------------------|
| 商標登録を受けようとする商標 | 商標登録願、団体商標登録願、地域団体商標登録願、防護標章登録願 |
| 図面（項目名不定） | 意匠登録願の【○○図】（1項目1イメージ） |
| 図面代用写真（項目名不定） | 意匠登録願の【○○図】（1項目1イメージ） |
| 説明図 | 特徴記載書の【説明図】（1イメージのみ） |

### 3.3 図番号の枝番

【図】【表】【化】【数】には枝番（サブ番号）を付与可能。
使用可能な記号: 英数字、`.`（ピリオド）、`(` `)`、`-`（ハイフン）

## 4. 文字修飾・改行

**出典**: `03_09_mojisyusyoku-kaigyo.pdf`

### 4.1 使用可能な装飾

| 書類種別 | 文字修飾 | 改行 |
|---------|---------|------|
| **XML系** | 半角 / 下線 / 上付 / 下付 | ○ |
| **SGML系** | 半角 / 倍角 / 下線 / 上付 / 下付 | ○ |

> **参考**: 手続補正書・誤訳訂正書等で【発明の名称】【考案の名称】を補正する場合は **下線のみ** 使用可能。

### 4.2 XML系での適用箇所

| 標準項目 | 適用範囲 |
|---------|---------|
| 請求項 | 【請求項ｎ】 |
| 段落番号 / 図の説明 | 明細書【０００１】、【図面の簡単な説明】内の【図ｎ】（【図ｎ】は改行不可） |
| 配列表 | 配列表（ST.25） — **改行のみ可、文字修飾は不可** |
| 要約書 | 要約書の内容 |
| 段落 | 外国語明細書 / 請求の範囲 / 図面 / 要約書 |
| 意見の内容 | 各種意見書・上申書・弁明書 等 |
| 添付物件 | 限定書類のみ |
| 補正の内容 | 手続補正書・誤訳訂正書 等 |
| 証明に係る事項 | 証明請求書 |

### 4.3 PCT国際出願（英語）の場合

| 標準項目 | 適用範囲 |
|---------|---------|
| Claim | [Claim n] |
| Title of Invention / 段落番号 / 図の説明 | [Title of Invention]、[000n]、[Brief Description of Drawings] 内の [Fig. n]（[Title]と[Fig. n]は改行不可） |
| 要約書 | 要約書の内容 |

> **強制改行**: 改行が使える箇所で半角約10,000文字以内に改行がない場合、強制改行される。改行位置を調整したい場合はHTML上で事前に改行を入れる。

## 5. 段落番号の記載方法

**出典**: `03_09_danraku.pdf`

### 5.1 基本ルール（特許願・実用新案登録願・PCT国際出願日本語）

> ⚠️ **段落番号は必ず「０００１」からの連番** にすること。

段落番号の中にだけ、【特許文献ｎ】【非特許文献ｎ】【化ｎ】【数ｎ】【表ｎ】【図ｎ】等の項目を記載できる。

### 5.2 記載可否一覧

| 標準項目 | 項目名 | 文字 | 改行 | イメージ | 文字修飾 | リンク |
|---------|--------|:----:|:----:|:--------:|:--------:|:------:|
| 段落番号 | 【０００ｎ】 | ○ | ○ | ○ | ○ | × |
| 特許文献 | 【特許文献ｎ】 | ○ | × | × | × | × |
| 非特許文献 | 【非特許文献ｎ】 | ○ | × | × | × | × |
| 化学式 | 【化ｎ】 | × | × | ○※1 | × | × |
| 数式 | 【数ｎ】 | × | × | ○※1 | × | × |
| 表 | 【表ｎ】 | × | × | ○※1 | × | × |
| 図の説明 | 【図ｎ】※2 | ○ | × | × | ○ | × |

- ※1: イメージは1枚だけ挿入可能
- ※2: 【図ｎ】を記載できるのは【図面の簡単な説明】の中だけ

### 5.3 重要な注意

- 化学式・数式・表を **文字や罫線素片で記載する場合は、【化ｎ】【数ｎ】【表ｎ】を記載してはいけない**

## 8. MemoriaKids書類への適用チェックリスト

本特許出願（MK-PAT-2026-001）の書類を電子出願仕様に準拠させるための確認項目。

### 8.1 PDFファイル全般

- [ ] PDFバージョン 1.2〜1.7 で出力されているか
- [ ] 用紙サイズが A4 縦（21×29.7cm）になっているか
- [ ] PDFのプロパティに **タイトル設定がない** か（あれば削除）
- [ ] PDFに **パスワード・暗号化が設定されていない** か
- [ ] PDFにコメント・注釈が **含まれていない** か
- [ ] フォントがすべて埋め込まれているか

### 8.2 図面（図1〜図3）

- [ ] PNG形式（JPEGではない）
- [ ] **モノクロ2値**（グレースケールやカラーではない）
- [ ] 透過（アルファチャンネル）情報なし
- [ ] アニメーション形式ではない
- [ ] 解像度: 200/300/400 dpi いずれかで、ドット数が以下のいずれかに収まる
  - 200dpi: 1338 × 2007 ドット以下
  - 300dpi: 2007 × 3011 ドット以下
  - 400dpi: 2677 × 4015 ドット以下
- [ ] mm換算で **170 × 255mm 以内**
- [ ] 各図に通し番号【図１】【図２】【図３】が付与されているか

### 8.3 明細書

- [ ] 段落番号が **【０００１】から連番** で全角数字
- [ ] 段落番号は『発明の詳細な説明』内のみで使用
- [ ] 化学式・数式・表を文字で記載した箇所には【化ｎ】【数ｎ】【表ｎ】を使用しない
- [ ] 【図ｎ】の参照は【図面の簡単な説明】の中で行う
- [ ] 文字修飾は半角・下線・上付・下付のみ
- [ ] 文字コードは JIS-X0208-1997 範囲内（機種依存文字 ①②㈱ などは使わない）

### 8.4 特許願

- [ ] 整理番号は英大文字・数字・ハイフン・10桁以内（`MK-PAT-2026-001` → 14文字なのでNG、要短縮）

  > ⚠️ **修正必要**: 現在の整理番号 `MK-PAT-2026-001` は14文字で10桁制限を超えている。例: `MK26001` 等に短縮要。

- [ ] 識別番号は数字9桁固定（申請人利用登録で取得）
- [ ] 電話番号は半角・20桁以内・記号は `( ) - / : . , ;` のみ
- [ ] 予納台帳番号は数字6桁固定
